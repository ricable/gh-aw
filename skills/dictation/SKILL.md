## Fix Speech-to-Text Errors

Common speech-to-text misrecognitions and their correct forms:

### Configuration Terms
- "safe out puts" → safe-outputs
- "safe in puts" → safe-inputs
- "front matter" → frontmatter
- "tool set" → toolset
- "MCP server" → mcp-server (lowercase)
- "cache memory" → cache-memory
- "lock file" → lockfile or .lock.yml

### Engine Names
- "co pilot" → copilot
- "clawed" → claude
- "code x" → codex

### GitHub Concepts
- "workflow dispatch" → workflow_dispatch
- "pull request" → pull_request (in YAML), pull-request (in prose)
- "at copilot" → @copilot

### Commands
- "GH AW" → gh-aw or gh aw (depending on context)
- "recompile" → recompile (not re-compile)
- "MCP inspect" → mcp inspect

### Common Homophone Errors
- "their" vs "there" vs "they're" - use context
- "its" vs "it's" - use "its" for possessive, "it's" for "it is"
- "your" vs "you're" - use "your" for possessive, "you're" for "you are"

### Hyphenation
- safe-outputs (hyphenated)
- safe-inputs (hyphenated)
- cache-memory (hyphenated)
- agentic-workflows (hyphenated when used as compound modifier)
- workflow run (two words, no hyphen)
- lock file → lockfile (one word) or .lock.yml (file extension)

### Capitalization
- GitHub (capital G and H)
- MCP (all caps when abbreviation)
- YAML (all caps)
- JSON (all caps)
- API (all caps)

## Clean Up and Improve Text

When cleaning up dictated text:

1. **Remove filler words**: humm, you know, um, uh, like, basically, actually, sort of, kind of, I mean, well, so, right, okay
2. **Remove repetitive phrases**: "as I said", "like I mentioned", "you know what I mean"
3. **Fix run-on sentences**: Break long sentences into shorter, clearer ones
4. **Improve clarity**: Replace vague terms with specific technical terms
5. **Use active voice**: Prefer "The workflow compiles" over "The workflow is compiled"
6. **Be concise**: Remove unnecessary words while preserving meaning
7. **Maintain technical accuracy**: Keep domain-specific terms intact
8. **Fix grammar**: Correct subject-verb agreement, tense consistency
9. **Professional tone**: Remove casual language like "gonna", "wanna", "kinda"
